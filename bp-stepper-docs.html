bp-stepper 以 Custom Element 語法驅動的步驟指示元件

支援橫向與縱向排版、多列換行、三種連接線樣式、十種色彩主題、可點擊互動與 Custom Events，所有視覺細節皆可透過 data-* 屬性調整。

安裝

將 bp-stepper.js 放在專案目錄，於 HTML 中引入一次即可。

<script src="bp-stepper.js"></script>
不依賴任何外部函式庫，可與 Bootstrap 或其他 CSS 框架共存。

自動編號

預設開啟。元件初始化時插入 <span class="bp-step-badge">，依序顯示 1、2、3⋯。狀態切換時自動更新為 ✓ 或 !。

設定 data-auto-number="false" 關閉。關閉後需自行放置 <span class="bp-step-badge">，或不放置（無徽章）。

<bp-stepper data-auto-number="false">
  <bp-step>
    <span class="bp-step-badge">A</span>
    <bp-step-title>選項 A</bp-step-title>
  </bp-step>
</bp-stepper>

排列方向

預設橫向。設定 data-layout="vertical" 切換縱向，連接線與箭頭方向自動調整。

vertical
收到訂單 備貨中 出貨 已送達

data-layout="vertical" 與 data-wrap 不可同時使用。

多列換行

設定 data-wrap="N"，前 N 個步驟排在第一列，第 N+1 個起排在第二列（靠左對齊）。兩列之間由 ∪ 形連接器橋接，箭頭指示流向。最多支援兩列。

第一列最後一步的狀態會同步到 ∪ 形連接器的顏色。

data-wrap="4"，共七個步驟，點擊可切換狀態
需求訪談 競品分析 資訊架構 視覺設計 原型測試 開發交付 上線驗收 
<bp-stepper data-wrap="4" data-theme="special">
  <bp-step data-state="done">…</bp-step>
  <bp-step data-state="done">…</bp-step>
  <bp-step data-state="done">…</bp-step>
  <bp-step data-state="active">…</bp-step>
  <bp-step>…</bp-step>
  <bp-step>…</bp-step>
  <bp-step>…</bp-step>
</bp-stepper>

DOM 重組說明

元件初始化時會自動把 HTML 裡的 <bp-step> 重新分組，建立以下結構：

<bp-stepper data-wrapped>           <!-- data-wrapped 由元件自動加上 -->
  <div class="bps-row">          <!-- 第一列：步驟 1..N -->
    <bp-step>…</bp-step>
  </div>
  <div class="bps-u-turn"></div>  <!-- ∪ 形連接器 -->
  <div class="bps-row">          <!-- 第二列：步驟 N+1.. -->
    <bp-step>…</bp-step>
  </div>
</bp-stepper>

∪ 形連接器顏色

∪ 形連接器的顏色跟隨第一列最後一步（第 N 步）的狀態，由 setActive、setError、resetStates 自動同步。

第 N 步狀態∪ 形連接器顏色
（預設）主色 --bps-color
active進行中強調色 --bps-color-active
done完成色 --bps-color-done
error錯誤色 --bps-color-error

步驟數不足時

若總步驟數 ≤ N，換行不會觸發，元件以正常單列橫向呈現。

data-wrap 與 data-layout="vertical" 不可同時使用。
bp-stepper[data-wrapped] 的寬度由 CSS 設定為 fit-content，若父元素有明確寬度限制，請在外層加一個 overflow-x: auto 的容器。

可點擊步驟

設定 data-clickable="true"，點擊步驟後立即更新狀態。搭配 data-wrap 時，兩列的步驟皆可點擊，∪ 形連接器顏色同步更新。

鍵盤：Tab 移動焦點，Enter 或 Space 觸發。

點擊任一步驟
規劃需求與時程 設計原型與評審 開發實作與測試 上線部署與監控 
<bp-stepper data-clickable="true">…</bp-stepper>

JavaScript API

所有方法掛載在 window.BPStepper。啟用 data-wrap 時，index 仍以所有步驟的扁平順序計算。

BPStepper.setActive(el, index)

將指定步驟設為 active，前方自動 done，後方清除狀態。index 從 0 起算。index >= 步驟數 時全部設為 done 並觸發 bps:complete。

const el = document.querySelector('bp-stepper');
BPStepper.setActive(el, 1);   // 第二步設為 active
BPStepper.setActive(el, 99);  // 全部完成

BPStepper.setError(el, index)

將指定步驟設為 error，不影響其他步驟。∪ 形連接器同步更新。

BPStepper.setError(el, 2);

BPStepper.resetStates(el)

清除所有步驟狀態，徽章恢復數字，∪ 形連接器恢復主色。

BPStepper.resetStates(el);

BPStepper.init(root)

對 root（或 document）內所有 <bp-stepper> 重新初始化。用於動態插入 HTML 的場合。

container.innerHTML = `<bp-stepper data-wrap="3">…</bp-stepper>`;
BPStepper.init(container);
正常插入 DOM 的 <bp-stepper> 會透過 connectedCallback 自動初始化，通常不需要手動呼叫 init()。

自訂事件

事件名稱觸發時機detail 內容
bps:change每次 setActive 切換步驟{ index, prevIndex, el }
bps:complete所有步驟變為 done{ el }
點擊步驟觀察事件
步驟一 步驟二 步驟三
等待事件…
const el = document.querySelector('bp-stepper');

el.addEventListener('bps:change', (e) => {
  const { index, prevIndex } = e.detail;
  console.log(`從 ${prevIndex} → ${index}`);
});

el.addEventListener('bps:complete', () => {
  console.log('全部完成！');
});

全域預設

在 bp-stepper.js 載入後、HTML 解析前修改 BPStepper.defaults，即影響整頁所有步驟器。

BPStepper.defaults.padTop    = '8px';
BPStepper.defaults.theme     = 'shell';
BPStepper.defaults.cardWidth = '260px';
全域預設只影響尚未初始化的步驟器。建議在 <script src="bp-stepper.js"> 之後立即修改，早於頁面內容。
BPStepper.BRAND.lavender   // '#C3A5E5'
BPStepper.THEMES.shell     // { color, colorActive, colorDone, … }
