首頁

這是首頁內容，掛載在各種不同類型的 HTML 容器元素上。

關於

menu-flex 元件可以掛載到任何能包含子節點的 HTML 元素。

文章

<article> 語意上代表獨立的內容區塊，非常適合用來承載有導覽的內容。

導覽

<nav> 通常用於頁面的導覽區域，掛載選單後形成雙層導覽結構。

側邊欄

<aside> 語意上代表與主內容相關但非主線的補充資訊。

詳細資訊

<details> 原生支援展開折疊，與 menu-flex 一起使用可形成雙層展開機制。

圖說

<figure> 通常包含圖片與圖說，也可以是程式碼區塊或引用內容的容器。

對話框

<dialog> 可做為 modal 的容器，配合 menu-flex 後形成含選單列的面板。

主內容

<main> 代表頁面的主要內容區，每頁建議只出現一次。

📦 menu-flex — 各種容器類型

只要不是 void element，任何 HTML 容器都可以作為 attach-to 目標

① <div> — 最常見的容器

<div> ✓ 完全支援

② <section> — 語意分區

<section> ✓ 完全支援 代表頁面中某個主題的獨立區塊

③ <article> — 獨立內容

<article> ✓ 完全支援 代表可獨立散佈的內容（部落格文章、新聞等）

④ <main> — 主內容區

<main> ✓ 完全支援 一頁建議只用一個

⑤ <aside> — 側欄補充

<aside> ✓ 完全支援 補充資訊、工具欄、widget 的理想容器

⑥ <nav> — 導覽區塊

<nav> ✓ 完全支援 形成選單選單的「雙層導覽」結構

⑦ <figure> — 圖文容器

<figure> ✓ 完全支援 圖片、程式碼區塊、引用的語意容器

⑧ <details> — 原生折疊

<details> ⚠ 可用，但原生 summary 會排在選單下方

選單列會 prepend 進去排在最前，原有的 <summary> 子節點會往後移。

（原生摘要，已隱藏）

⑨ <dialog> — Modal 面板

<dialog> ⚠ 可用，需用 CSS 開啟（取消 UA 預設 display:none）

原生 <dialog> 預設被瀏覽器隱藏，需透過 dialog { display:flex; } 或 JS .showModal() 開啟。此展示直接用 CSS 覆寫。

⑩ 容器支援度總表

元素語意用途支援狀態注意事項
<div> 通用容器 ✓ 完全支援 —
<section> 主題分區 ✓ 完全支援 —
<article> 獨立內容 ✓ 完全支援 —
<main> 主內容區 ✓ 完全支援 一頁建議只用一個
<aside> 側欄補充 ✓ 完全支援 —
<nav> 導覽區塊 ✓ 完全支援 —
<header> 頁首區塊 ✓ 完全支援 —
<footer> 頁尾區塊 ✓ 完全支援 —
<figure> 圖文容器 ✓ 完全支援 —
<form> 表單容器 ✓ 完全支援 —
<details> 折疊容器 ⚠ 可用 原有 <summary> 子節點會排在選單後面
<dialog> Modal 面板 ⚠ 可用 需用 CSS 或 JS 開啟，並覆寫 UA 樣式
<table> 表格 ✗ 不建議 瀏覽器對 table 子元素限制嚴格，會破壞結構
<img> 等 void自閉合元素 ✗ 不支援 無法包含子節點；元件會輸出 console.warn 並跳過
